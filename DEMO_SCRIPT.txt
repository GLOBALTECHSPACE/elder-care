[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how a query is processed. 
First, every input starts at the Security Checkpoint node. If any PII like a Medicare card ID or phone number is found, it gets scrubbed automatically; if a prompt injection or financial keyword is detected, it gets routed to the Security Violation Handler to protect the user.
Clean inputs pass to the Run Orchestrator node. The orchestrator agent relies on three specialized sub-agents: the Medication Agent, the Health Metrics Agent, and the Cognitive Games Agent. These sub-agents retrieve and store records by talking directly to a local Model Context Protocol server.
Finally, if a caregiver requests to record a log, the orchestrator triggers a Human-In-The-Loop approval pause. Only after the caregiver manually confirms 'yes' does the system save the record, routing the result to the Final Output. (pause)
